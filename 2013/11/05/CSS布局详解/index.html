CSS定位详解

CSS有三种基本的定位机制：普通流、浮动和绝对定位

　　除非专门制定，否则所有元素都在普通流中定位。也就是普通流中的元素的位置由元素在HTML中的位置决定。
　　块级元素（如：div、ul、ol、p、h）从上到下一个接一个地排列，框之间的垂直距离是由框的垂直外边距计算出来。
　　行内元素（如：a、img、input、span）在一行中水平布置。可以使用水平内边距、边框和外边距调整它们的间距。但是，垂直内边距、边框和外边距不影响内框的高度。不过设置行高可以增加行框的高度。

　　下面具体介绍浮动和position定位：
　　浮动
　　float 属性定义元素在哪个方向浮动。在 CSS 中，任何元素都可以浮动。浮动元素会生成一个块级框，而不论它本身是何种元素。由于浮动的框不在文档的普通流中，所以文档普通流中的其它元素表现的就像该浮动框不存在一样。
　　position定位
　　定位算是CSS中难以理解的属性之一吧。希望看完本文后您对position有更清晰全面的了解。
　　position属性有四个属性值:
　　　　1.relative
　　　　2.absolute
　　　　3.fixed
　　　　4.static(默认值)
　　下面分别讲述这四个属性值。

　　1.relative（相对定位）
　　首先，要搞清楚相对定位是相对哪个对象来进行偏移的。答案是：它本身在普通流中的位置，基于对象margin的左上角。如果对一个元素进行相对定位，通过设置水平或垂直位置，让这个元素“相对于”其正常位置进行移动。元素位置通过“left”,“top”,“right”,“bottom” 属性进行定位。
　　注意：使用相对定位时，无论元素是否进行移动，元素仍占占据原来的空间。该元素周围的元素位置不会因此受到影响。所以，移动元素可能会导致它覆盖其他元素。

　　2.absolute（绝对定位）
　　同样，先要说明绝对定位是相对哪个对象来进行偏移的。答案是：绝对定位的元素的位置是相对于最近的已定位的祖先元素（祖先元素的定位可以是relative/absolute/fixed），如果元素没有已定位的祖先元素,那么它的位置相对于以body为定位对象，按照浏览器的窗口进行定位。元素位置通过“left”,“top”,“right”,“bottom” 属性进行定位。
　　注意：设置为绝对定位的元素慧聪文档流中完全删除，就好像原来该元素根本不存在一样，该元素原来占的空间会关闭。所以，该元素移动后会，普通流中的其它元素布局就像绝对定位的元素不存在一样。因为绝对定位的元素与文档流无关，所以它们可以覆盖页面上的其它元素，可以通过设置z-index属性来控制这些元素的堆放次序。

　　3.fixed
　　生成绝对定位的元素，相对于浏览器窗口进行定位。元素位置通过“left”,“top”,“right”,“bottom” 属性进行定位。

　　4.static(默认值)
　　没有定位，元素出现在正常的流中（忽略top,bottom,right,left或者z-index声明）
