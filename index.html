原生js的ready和onload

我们知道js选取界面元素代码在执行时，相关的DOM必须是加载完成的。因此，当在head中写js代码时，必须添加把相关代码写在window.onload函数中如下：

window.onload = function(){
    //js代码....
};

window.onload是当界面中所有的资源全部加载完成后才执行的一个回调函数。所有的资源包括DOM元素、js、css、图片等等。问题是如果界面中的资源非常多，包括大量的图片和外部文件时该函数的执行会等到很久之后，用户体验和网站的性能都不是很可靠。用jQuery的童鞋都知道，jquery的代码要写在$(function(){//代码})中，此处代码就是在界面元素加载完执行的，功能上类似于window.onload函数。实际却是js中的ready事件,不同于onload的是：ready事件等DOM元素加载完就执行。

下面就详细介绍js的ready事件：

首先ready是一种描述，指的是界面DOM元素都加载完成的状态。针对不同的浏览器具体的ready事件不同。
高级浏览器即非IE6/7/8写法:

//高级浏览器    
document.addEventListener("DOMContentLoaded",function(){
    //ready完成执行代码
},false);

//IE6/7/8
document.onreadystatechange = function(){
    if(document.readyState == "complete"){
        //ready完成
    }
};

综上可以封装一个兼容的函数，类似jQuery中的ready，如下：

function $(fn){
    if(document.addEventListener){
        document.addEventListener("DOMContentLoaded",fn，false);
    }else{
        document.onreadystatechange = function(){
            if(document.readtState == "complete"){
                fn();
            };    
        };
    };
};

当使用时不用写window.onload,直接使用$(function(){//js代码});即可。

CSS定位详解

CSS有三种基本的定位机制：普通流、浮动和绝对定位

　　除非专门制定，否则所有元素都在普通流中定位。也就是普通流中的元素的位置由元素在HTML中的位置决定。
　　块级元素（如：div、ul、ol、p、h）从上到下一个接一个地排列，框之间的垂直距离是由框的垂直外边距计算出来。
　　行内元素（如：a、img、input、span）在一行中水平布置。可以使用水平内边距、边框和外边距调整它们的间距。但是，垂直内边距、边框和外边距不影响内框的高度。不过设置行高可以增加行框的高度。

劝学

一篇《劝学》作为第一篇博文，与大家共勉。学无止境，学习不止。

　　君子曰：学不可以已。青，取之于蓝，而青于蓝；冰，水为之，而寒于水。木直中绳，輮以为轮，其曲中规，虽有槁暴，不复挺者，輮使之然也。故木受绳则直，金就砺则利，君子博学而日参省乎己，则知明而行无过矣。故不登高山，不知天之高也；不临深溪，不知地之厚也；不闻先王之遗言，不知学问之大也。干、越、夷、貉之子，生而同声，长而异俗，教使之然也。诗曰：「嗟尔君子，无恒安息。靖共尔位，好是正直。神之听之，介尔景福。」神莫大于化道，福莫长于无祸。
